# A 组模块实现说明与代码结构

本文档说明本次在 `document/A` 对应四个模块（数据分布、副本管理、负载均衡、本地存储引擎）的实际落地内容、主流程接入方式和测试方法。

## 1. 本次完成了什么

### 1.1 数据分布模块

- 新增 `DistributionManager` 抽象与 `DistributionManagerImpl` 实现。
- 新增 `ConsistentHashRing`（支持虚拟节点）用于 key -> node 选择。
- 新增 `RoutingTable`（带 TTL）减少重复元数据查询。
- 支持 `onClusterChange` 触发哈希环重建与缓存失效。

### 1.2 副本管理模块

- 新增 `ReplicaManager` / `ReplicaManagerImpl`。
- 新增三段式职责拆分：
  - `PrimaryHandler`：主节点写入与副本确认；
  - `ReplicaHandler`：读请求副本选择；
  - `FailoverHandler`：主节点故障后的主备切换与元数据更新。
- 新增 `ReplicaSyncTransport` 作为 RPC 同步适配点，便于后续替换真实网络实现。

### 1.3 负载均衡模块

- 新增 `LoadBalancer` / `LoadBalancerImpl`。
- 新增 `WeightedRoundRobin` 实现读请求加权轮询。
- 新增 `NodeMonitor` + `MigrationManager` 形成负载上报与迁移触发链路。

### 1.4 本地存储引擎模块

- 在 `minisql-worker` 新增 `LocalStorageEngine` / `LocalStorageEngineImpl`。
- 新增 `PartitionManager` 负责分片文件导入导出与落盘。
- 新增存储模型：`Row`、`ScanOptions`、`ReplicationEntry`、`OpType`。
- 支持行级 CRUD、scan、副本日志回放、分片导入导出。

### 1.5 元数据底座（支撑四个模块）

- 新增 `NodeInfo`、`ClusterEvent`、`NodeLoad`。
- 新增 `MetadataService` 与 `ZkMetadataService`，用于主副本元数据持久化与读取。

## 2. 主流程接入说明

### 2.1 Worker 主流程接入

入口：`minisql-worker/src/main/java/com/zju/minisql/worker/WorkerStarter.java`

- 启动时初始化 `LocalStorageEngine`。
- 写入一条启动记录 `system::boot`，验证本地存储层可用。
- 与原有 RPC 服务注册流程并存，不影响旧联调路径。

### 2.2 Client 主流程接入

入口：`minisql-client/src/main/java/com/zju/minisql/client/demo/GroupBQueryDemo.java`

- 启动后会初始化 `ZkMetadataService` 并写入示例分片主副本映射。
- 创建并接线以下模块实例：
  - `DistributionManagerImpl`
  - `LoadBalancerImpl`
  - `ReplicaManagerImpl`
- `HashQueryRouter` 已增加可选 `DistributionManager` 注入能力：
  - 命中主键等值条件时，优先走 A 组分布模块路由结果；
  - 未命中时仍广播，保持兼容。

## 4. 模块测试脚本

新增脚本位置：`scripts/`

- `test-a-distribution.ps1`：数据分布模块测试
- `test-a-replica.ps1`：副本管理模块测试
- `test-a-loadbalance.ps1`：负载均衡模块测试
- `test-a-storage.ps1`：本地存储引擎模块测试

脚本默认在项目根目录执行 Maven 定向测试，便于单模块回归。

## 5. 当前限制与后续建议

- 迁移流程目前是控制面编排骨架，真实数据迁移 RPC 需要后续与 Worker 数据面进一步打通。
- 副本同步传输目前采用接口抽象，生产级实现应接入真实网络调用与超时重试。
- 建议下一步补齐：
  - 分片级统计采集（用于更精确迁移决策）
  - 副本同步进度持久化
  - 端到端故障注入测试（主节点宕机、副本恢复、迁移回滚）
